**Category:** enhancement
**Summary:** Add a mechanism to verify that AXL files respect their declared line-count budgets in AGENTS.md.

**Current behavior:**
`AGENTS.md` declares five load budgets: `router` (<=200), `base_rules` (<=300), `types` (<=150), `domain_overlay` (<=200 each), and `ledger_summary` (<=150). No mechanism checks or enforces these. The `ledger_summary` budget was added without a clear definition of which files it covers.

**Desired behavior:**
A script under `bin/` reads the budget declarations from `AGENTS.md` and checks actual file line counts against them. It exits non-zero on violation. The `ledger_summary` scope is clearly defined.

**Key interfaces:**
- The script reads the `@LOAD_BUDGET` block from `AGENTS.md` to discover declared budgets
- Budget-to-file mapping: `router` -> `AGENTS.md`, `base_rules` -> `rules/base.axlr`, `types` -> `axl/types.axlt`, `domain_overlay` -> `rules/*.axlr` (excluding `base.axlr`), `ledger_summary` -> `ledgers/*.axls`
- Output: list of files with actual vs declared line counts; exit 0 if all within budget, exit 1 if any violation

**Acceptance criteria:**
- [ ] A script exists under `bin/` that checks line counts against declared budgets
- [ ] The check covers all five budget categories
- [ ] `ledger_summary` scope is clearly defined (which files count)
- [ ] Running the script on the current repo passes (or reports real violations)
- [ ] Script is executable
